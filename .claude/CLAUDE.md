## Bash コマンドのルール

- `|`、`&&`、`||`、`2>/dev/null` などを含む**複合コマンドは使わない**
- パターンマッチが効かないため、allow ルールをすり抜けて毎回確認が出る
- 複数の処理が必要な場合は、**個別のツール呼び出しに分割する**

例：
- NG: `cat file | python3 -m json.tool 2>/dev/null || echo "not found"`
- OK: Read ツールでファイルを読む → 必要なら別途 Bash で個別実行
